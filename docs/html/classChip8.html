Detailed Description

A chip8 emulator.

It stores emulation state and provides an interface for running 2 byte chip8 instructions. You can either run a chip8 program with it by calling Chip8::load_program and then repeatedly calling Chip8::run_program_instr, or run single instructions using Chip8::run_instr.

While running the program, you can emulate pressing key on real console by calling Chip8::press_key (there are 16 keys, each of them corresponding to 4 bit number from 0x0 to 0xf) and then "see" produced graphics with Chip8::get_screen, which returns current pixel array, a 2D array of booleans.

Note that there are no public, helpful in unit tests, functions mapping some expected functionality to 2 byte instruction (jump(addr) for instance). You have to create the instruction code yourself and then call Chip8::run_instr. This is intentional, as it greatly simplifies instruction parsing logic, which is just a mapping between first byte of instruction and some private callback (using std::array) without really causing much trouble.
